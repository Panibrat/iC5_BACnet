Логика изменение состояния в UI."Путь переменной".

1. Контроллер изменил состояние переменной 
2. Сервер прочитал новое состояние, 
сравнил с тем что есть в буффере, 
принял решение поменять в значение в базе данных, 
отослал запрос в базу данных,
получил подтверждение от базы данных,
отправил сокет-ивент 'есть новые данные' на клиент.
3. Клиент обработал сокет-ивент, 
запустил экшн "получить новые данные с базы", 
отправил на сервер GET-запрос.
4. Сервер обработал  GET-запрос, 
послал в базу данных запрос о новом состоянии,
получил новое состояние,
ответил на GET-запрос новым состоянием.
5. Клиент получил новое состояние,
изменил store
6. UI обновился
